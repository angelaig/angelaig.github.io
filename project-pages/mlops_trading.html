MLOps Pipeline for Trading Models
Freelance Project
MLOps pipeline for trading models with drift detection, leveraging Apple-level engineering experience. Trains a baseline LightGBM model for return prediction, then builds full production infrastructure around it : including feature stores, model registry, automated retraining, and champion/challenger deployment.
Project Details / Background
Designed and built an end-to-end MLOps pipeline for trading models that addresses the unique challenge of financial model degradation. Unlike traditional ML systems, financial models degrade due to regime changes, evolving market microstructure, and shifting correlations : this pipeline was built to detect and respond to that drift automatically.
Trained a baseline LightGBM model for return prediction, then constructed full production infrastructure around it. The pipeline includes a feature store that tracks feature versions and computation timestamps, ensuring reproducibility and auditability across all model training runs.
Integrated MLflow for model registry and experiment tracking, providing a centralised view of all model versions, hyperparameters, metrics, and artifacts. Every training run is logged and comparable, enabling data-driven decisions about model promotion.
Implemented drift detection using Population Stability Index (PSI) and Kolmogorov-Smirnov tests on input features to catch data drift early. For concept drift, the system monitors rolling Sharpe ratio and information coefficient decay : metrics that capture the financial relevance of model predictions over time.
Built automated retraining pipelines using Apache Airflow DAGs that trigger when drift thresholds are breached. The retraining workflow includes data validation, feature recomputation, model training, and evaluation against the current champion model before any promotion decision.
Developed a champion/challenger framework for shadow-testing new models before promotion to production. Challenger models run in parallel against live data, and their predictions are compared to the champion's performance across multiple financial metrics before a swap is approved.
The monitoring stack uses Grafana dashboards backed by Redis for real-time metric streaming and PostgreSQL for historical storage. EvidentlyAI generates drift reports that feed into Airflow triggers, closing the loop between detection and automated response.
